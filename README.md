# 🍉 敲瓜听声 · 频率分析 (MVP)

点按钮录音 → 敲击西瓜 → 停止后自动分析主频,判断生熟。**纯前端单文件,无后端**,直接发链接就能用。

## 原理

敲击 = 给西瓜一个脉冲激励,它以自身**共振频率**震动。熟瓜内部组织变软有空隙,声音低沉(主频偏低);生瓜致密清脆(主频偏高)。程序流程:

```
录整段 PCM → 定位最响的一下敲击 → 加 Hann 窗 → FFT → 找主频/质心
```

- 录音走 Web Audio 拿**原始 PCM(无损)**,不用 mp3/aac(有损压缩会破坏频率成分)。
- FFT 为自实现的迭代 Cooley-Tukey,已用已知频率正弦波验证,误差 ±0.01Hz。
- 抛物线插值让主频精度突破 FFT 频率分辨率(bin)的限制。

## 参考阈值(学术共识,仅作起点)

| 状态 | 主频 |
|------|------|
| 过熟 | < 130 Hz |
| **熟瓜** | **130 – 160 Hz** |
| 偏生 | > 160 Hz |

来源:中科院声学所(133–160Hz)、MDPI 2026、Zhang 2019。

⚠️ **绝对 Hz 值会因手机麦克风、敲击力度、位置而整体漂移,别直接照搬。** 页面内置"标记生瓜/熟瓜"自校准:攒几个已知瓜的样本后,它自动按**你自己设备**的数据算分界线,判读也会优先用你的分界。

## 部署到 GitHub Pages(发链接)

`getUserMedia` 录音强制要求 HTTPS,GitHub Pages 自带证书,是理想选择。

```bash
# 1. 在 GitHub 建一个 repo, 比如 melon-tap, 然后在项目目录:
git init
git add index.html README.md
git commit -m "feat: 敲瓜听声 MVP"
git branch -M main
git remote add origin https://github.com/<你的用户名>/melon-tap.git
git push -u origin main
```

2. 打开 repo → **Settings → Pages** → Source 选 `main` 分支 `/ (root)` → Save。
3. 等 1 分钟,访问 `https://<你的用户名>.github.io/melon-tap/`。手机扫码/发链接即可用。

> 也可用 Vercel / Cloudflare Pages,拖一个 `index.html` 上去同样自带 HTTPS。

## 使用建议

- **iOS/Safari**:必须点按钮触发录音(已适配),首次会弹麦克风授权。
- **敲击一致性**:同一位置、同样力度,同一个瓜敲 3 次取平均更稳。
- **环境安静**:关掉了浏览器的降噪/AGC 以保真,但环境噪声大仍会干扰,尽量安静处测。
- **先校准**:测真瓜前,先敲几个切开验证过生/熟的瓜建立你的基准线。
