# 🍉 敲瓜听声 (Melon Tap)

> 敲一下西瓜，听声音判断它是**生瓜 / 好瓜 / 老瓜**——纯前端网页，打开就能用，不上传、不联网、无后端。

一个把"敲西瓜挑熟"这件民间玄学，用**声学 + 傅里叶变换**做成可视化的小玩具。手机浏览器打开，点录音、敲西瓜，它会画出波形、频谱，并算出主频给个判断。

## ✨ 在线体验

👉 **https://xxyykk.github.io/melon-tap/**

用手机浏览器打开（iOS 用 Safari），允许麦克风即可。所有计算都在你手机本地完成，录音不会上传到任何服务器。

## 🔬 原理：为什么敲声能判断生熟？

敲击西瓜，相当于给它一个瞬间的能量激励，瓜会以自身的**共振频率**振动——就像敲不同的鼓音高不同。

- **生瓜**：内部组织致密紧实 → 声音清脆 → 主频**偏高**
- **好瓜**：组织成熟、含水足 → 声音饱满低沉 → 主频**居中**
- **老瓜**：组织变软、出现空腔 → 声音发闷 → 主频**偏低**

程序做的事就是把这段"嘭"声，从**时域**（振幅随时间）通过 **FFT（快速傅里叶变换）** 转到**频域**（能量随频率），找出能量最集中的那个频率（主频）。

```
录音 → 定位最响的一下敲击 → 截取瞬间片段加窗 → FFT → 找主频 → 对照区间判断
```

### 参考区间（学术共识，仅作起点）

| 类别 | 主频参考 |
|------|---------|
| 老瓜（过熟） | < 130 Hz |
| **好瓜** | **130 – 160 Hz** |
| 生瓜 | > 160 Hz |

> 来源：中科院声学所（成熟瓜 133–160 Hz）、MDPI 2026、Zhang 2019 等公开研究。
>
> ⚠️ **绝对频率值会因手机麦克风、敲击力度和位置而整体漂移，别照搬。** 建议同一个瓜、同一位置多敲几次看结果是否稳定，横向对比几个瓜的相对高低比单看绝对值更靠谱。

## 🖥️ 功能

- 🎤 **一键录音**，实时滚动波形（心电图风格，敲一下立刻看到尖峰）
- 📈 **波形图（时域）**：横轴是时间，回放动画 + 游标扫过，绿色高亮标出实际参与分析的窗口
- 📊 **频谱图（频域）**：横轴是频率，标出主频，三色背景带对应老/好/生区间
- 🔢 主频、余韵/衰减、频谱质心、敲击次数等指标
- 💬 **人话建议**：综合主频 + 余韵，直接告诉你能不能买、为什么
- 🔔 **材质兜底**：敲的明显不是西瓜时，改给"可能是金属/木头…"的材质倾向推测
- 🔒 **隐私优先**：录音全程仅在当前页面内存分析，不上传、不落盘、不联网，关掉页面即消失

## 🛠️ 技术实现

- **纯静态单文件**：全部逻辑在一个 `index.html` 里，无框架、无依赖、无构建。
- **录音**：Web Audio API，取**原始 PCM**（不用 mp3/aac——有损压缩会破坏要分析的频率成分）。
- **FFT**：自实现的迭代 Cooley-Tukey 算法，配合 Hann 窗 + 补零 + 抛物线插值提高主频精度。已用已知频率正弦波验证，误差 ±0.01 Hz。
- **可视化**：Canvas 2D 手绘波形与频谱，`requestAnimationFrame` 驱动动画。

关于"分析窗口"：敲击是**瞬态信号**（能量几十毫秒内衰减），所以只截取敲击瞬间约 120 ms 做 FFT，信噪比最高。窗口越长频率分辨率越高，但会混入余韵和噪声——120 ms 是这个场景的平衡点，页面上可自行调整。

## 🚀 本地运行 / 自行部署

因为是纯静态页面，任何静态服务器都行。注意麦克风录音**强制要求 HTTPS**（或 `localhost`）。

```bash
# 本地预览
git clone https://github.com/XxYyKk/melon-tap.git
cd melon-tap
python3 -m http.server 8000
# 浏览器打开 http://localhost:8000
```

部署到 GitHub Pages：仓库 **Settings → Pages → Source 选 `main` 分支 `/(root)`**，一分钟后即可通过 `https://<用户名>.github.io/melon-tap/` 访问（自带 HTTPS）。也可拖到 Vercel / Cloudflare Pages。

## 📱 使用建议

1. **iOS 用 Safari**，微信内置浏览器录音常受限，建议用系统浏览器。
2. **敲击一致**：同一位置、同样力度，一个瓜多敲几次更稳。
3. **环境安静**：页面关掉了浏览器降噪/AGC 以保真，噪声大会干扰结果。
4. **相对比较**：绝对频率会因设备漂移，多敲几个瓜看相对高低，比盯着一个绝对数值更靠谱。
